# 设计模式常用的七大原则

1. 单一职责原则
2. 接口隔离原则
3. 依赖倒转原则
4. 里氏替换原则
5. 开闭原则（ocp原则）
6. 迪米特法则
7. 合成复用原则

工厂模式就使用了ocp原则，

- **单一职责原则**：在类层面，一个类只负责一项职责；方法层面，一个方法只负责一项职责。
- **接口隔离原则**：一个类对另一个类的依赖应该建立在最小接口上。
